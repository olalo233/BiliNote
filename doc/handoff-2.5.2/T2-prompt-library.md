# T2 — 提示词小抄（备注区模板库）

## 背景

备注（extras）在提示词构建里是**原样追加到最末尾**（`prompt_builder.generate_base_prompt`：
BASE_PROMPT + 格式函数 + 风格片段 + extras）。用户按视频类型切换整段人格（做饭主厨/
技术教程/播客…），需要保存/复用能力。交互已用 live demo 与用户对齐（见下"交互定稿"）。

## 需求

### 后端

- 新增 `app/services/prompt_library_manager.py`，照 `cookie_manager.py` 模式：
  存 `config/prompts.json`，结构 `[{"name": str, "content": str}]`，实时读写、挂载卷持久化。
- 路由（并入 config router 或新 `prompt.py`）：
  - `GET /api/prompts` 列表
  - `POST /api/prompts` 新增/覆盖同名（body: name, content）
  - `DELETE /api/prompts/{name}`
- 首次无文件时以**预置官方模板**初始化（内容见下"预置模板"），用户可删。

### 前端（NoteForm 备注区）

交互定稿（与用户逐点确认）：
- 备注 label 行右侧加两个按钮：「提示词库」「存为模板」。
- **提示词库**：点开展开面板，顶部模糊搜索框（对**名称+内容**双字段匹配，实时显示"匹配/总数"），
  下方**固定高度列表**（约 5 行高，超出滚动，**不翻页**）；每行：名称 + 单行内容预览，
  行尾「载入」；悬停行显示「删除」。
- **载入 = 用模板内容替换整个备注框**（不是插入片段）。
- **存为模板**：点开小表单（名称输入 + 保存/取消），把当前备注框内容入库。
- 打开库面板时自动清空搜索、聚焦搜索框。
- 空库 / 无匹配有对应空态文案。
- 服务层 `services/prompt.ts`，与现有 axios 封装/`suppressToast` 一致。

### 预置模板（初始化写入）

至少含：`做饭主厨`、`技术教程 · 两段式`、`播客访谈`、`论文精读`。内容可参考用户现用的
"# Role 明星主厨…"与"输出分为两部分…"，写成通用可复用版本（不要塞入某个具体视频的信息）。

## 验收标准

1. 后端四个操作 curl 实测：GET 初始返回预置模板；POST 新增后 GET 可见；同名 POST 覆盖；
   DELETE 后 GET 不见；`config/prompts.json` 内容与操作一致（贴文件）。
2. 前端 `pnpm dev` 实操录证：搜索能按名称和内容命中并显示计数；列表超高度出现滚动；
   载入后备注框被替换；存为模板后出现在库顶部并持久化（刷新页面仍在）；删除生效。
3. 载入某模板后真实生成一篇笔记，确认该提示词实际进入了最终 prompt（笔记风格符合模板意图）。
4. `pnpm lint`（改动文件无新增 error）+ `pnpm build`；全量 pytest 通过。
